Running the JDBC4 Tests

Currently, there is one suite of tests used to verify JDBC4 features:

  • jdbc40 - These tests can run with either the embedded or network client, that is, under either the embedded or DerbyNetClient frameworks. In general, JDBC4 tests do not run the JCC client (DerbyNet framework) because that old client does not support the new JDBC4 feature set.

Verifying the JDBC4 feature set is a little tiresome right now. You must run the following commands in a shell window whose JAVA_HOME points at your jdk1.6 installation. After each suite run, check the test results. 

java com.splicemachine.dbTesting.functionTests.harness.RunSuite jdbc40

# make sure the above test run succeeded

java -Dframework=DerbyNetClient com.splicemachine.dbTesting.functionTests.harness.RunSuite jdbc40

# make sure the above test run succeeded
